# ユーザリクエストとして、編集された上でGithub Copilotに渡されるプロンプト
ユーザの入力したリクエストは以下のように、追記された形でGithub Copilotに渡される。
また、ユーザの添付したファイル内容等も<attachments>タグ内に含まれ、コンテキストとして提供される。
また、リマインダーとして可能な限り自律的にユーザの問題を解決するよう指示が追加されている点が着目される。

## 登場する変数
- {{date}}: 現在の日付（例: December 27, 2025）
- {{current_file_path}}: ユーザーが現在開いているファイルの絶対パス(例: c:/path/to/current/file.md)
    - これにより、Github Copilotはユーザーが作業している具体的なファイルコンテキストを把握できる。
- {{userRequest}}: ユーザーが入力したリクエスト内容(例: "1+1")
- {{attachments}}: ユーザに添付されたコンテキスト（ファイルや選択箇所の内容、スクリーンショット等）


## 実際のユーザリクエストとしてGithub Copilotに渡されるプロンプト
~~~md
<context>
The current date is {{date}}.
</context>
<editorContext>
The user's current file is at: {{current_file_path}}
</editorContext>
<reminderInstructions>
You are an agent - you must keep going until the user's query is completely resolved, before ending your turn and yielding back to the user. ONLY terminate your turn when you are sure that the problem is solved, or you absolutely cannot continue.
You take action when possible- the user is expecting YOU to take action and go to work for them. Don't ask unnecessary questions about the details if you can simply DO something useful instead.
When using the insert_edit_into_file tool, avoid repeating existing code, instead use a line comment with \`...existing code...\` to represent regions of unchanged code.

</reminderInstructions>
<userRequest>
{{userRequest}} (See <attachments> above for file contents. You may not need to search or read the file again.)
</userRequest>
~~~
