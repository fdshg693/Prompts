# 事前定義されたGithub Copilotに渡されるユーザープロンプト
Github Copilotが知っておくべきワークスペース固有の情報を与えている。
これらの情報を持つことで、ワークスペースの構造理解が可能となり、またターミナルコマンドの実行精度が上がる（bashなのかPowershellなのか、どのようなコマンドが事前インストールされていると想定されるか等）。

## 登場する変数
- {{workspace_trees}}: ユーザーのワークスペースツリーの情報
    - このツリー情報より、コンテキスト収集のために読むべきファイルを理解し、またそれらのファイルへのパスも把握します。
完全なツリーが毎回表示されるわけではなく、上限に達した場合は省略されます。
```markdown
1.txt
a/
	1.md
	aa/
		aaa/
			1.md
			2.md
		aab/
			1.json
b/
	ba/
		baa/
			1.md
			2.md
...
```

- {{os}}: ユーザーのOS情報（例: Windows, macOS, Linux）
- {{current_workspace}}: ユーザーが現在開いているワークスペースのルートフォルダへの絶対パス
    - ここで、絶対パスを渡すことで、ツール呼び出しの際の絶対パス指定の精度を向上させます。 

## 実際の事前定義済ユーザリクエストとしてGithub Copilotに渡されるプロンプト
~~~md
<environment_info>
The user's current OS is: {{os}}
</environment_info>
<workspace_info>
I am working in a workspace with the following folders:
- {{current_workspace}}
I am working in a workspace that has the following structure:
{{workspace_trees}}

This is the state of the context at this point in the conversation. The view of the workspace structure may be truncated. You can use tools to collect more context if needed.
</workspace_info>
~~~
