#　おすすめツール選択方法
まず、前提としてGithub Copilotに渡すツールはある程度制限をかけるべきです。不要なコンテキスト肥大を防ぎ、必要なツールを確実に使用させるためです。
そして、使用可能なツールをしっかりと把握することで、実際の指示にツールへの参照を組み込み思い通りのツール呼び出し動作・挙動を実現しやすくなります。

## あまり使わないツール一覧
以下のツールは基本的にツールセットに含める必要はありません。
- Notebook系ツール
    - Notebookに特化したツールであるため、NoteBookがワークスペース内に存在しなければ、わざわざツールに含める必要はありません。
- VS Code関連ツール
    - 特殊な用途で使用されるツールであり、VSCODE拡張機能の開発を行う場合や、VSCODEコマンド・VSCODE拡張機能インストールをAIに肩代わりさせたい場合（非常にレアケース）以外では、ツールに含める必要はありません。

## 非常によく使うツール一覧
以下のツールは重大な理由がない限り、ほぼ必ずツールセットに含めるべきです。

- ファイル操作系ツール
    - ファイル読み取り、ディレクトリ内容確認など、AIエージェントとして動作する上で基本的なファイル操作を行うために必要で、ほぼ必ず含めるべきです。
- ファイル編集・作成系ツール
    - ファイル編集等を行わせたくない場合を除き、ほぼ必ず含めるべきです。
- manage_todo_list
    - Github Copilotへのシステムプロンプトでも再三と言及されている、AIの挙動を制御するための重要なツールです。ほぼ必ず含めるべきです。
- get_errors
    - VSCODE上で発見可能な構文エラーや警告を取得するツールです。このツールにより、AIがコードの問題点を自動で修正して、より品質の高いコードを生成できるようになります。ほぼ必ず含めるべきです。

## よく使うツール一覧
よく利用されるため、ビルトイン以外のツールが多いなどの理由がない限り、ツールセットに含めることを推奨します。

- 検索・探索系ツール(file_search・grep_search・semantic_search)+get_search_view_results
    - コンテキスト取得のための強力なツール群であり、よく利用されるため、ツールセットに含めることを推奨します。
- fetch_webpage
    - 外部Webページから情報を取得するためのツールであり、最新の情報を取得する場合や、特定のWEBページの内容をコンテキストに含めたい場合に有用です。よく利用されるため、ツールセットに含めることを推奨します。
        - ただし、URLの内容を取得するだけであり、WEB検索を行うわけではないため、学習済URLおよびユーザーの与えたURLを超えてWEBアクセスが可能でないことに注意してください。
        - そのため、より高度なWEB探索が必要な場合は、Tavily MCPサーバー等を接続して、専用のツールを追加することを検討してください。
- サブエージェント系(runSubagent)
    - サブエージェントに処理を委譲することで、コンテキストを肥大化させず、非常に複雑で長い時間のかかるタスクを実行できる強力なツールです。非常に有用であるため、ツールセットに含めることを推奨します。
        - ただツールに含めるだけでは、そこまでGithub Copilotは利用しないようです。そのため、プロンプト等でこのツールを積極的に利用するように指示することを検討してください。
- タスク・ターミナル系(run_in_terminal・get_terminal_output)
    - ターミナルコマンドの実行や、タスクの実行結果取得を行うためのツールであり、ビルドやテスト、デプロイなどの操作を自動化する場合に有用です。よく利用されるため、ツールセットに含めることを推奨します。
- タスク・ターミナル系(create_and_run_task)
    - タスク作成に関しては単なるファイル編集ツールで代替可能ですが、タスクの実行を行えるため、そのプロジェクトでチェックしておきたいテストやフォーマッターの実行等をAIに任せたりチェックさせたりする場合に有用です。よく利用されるため、ツールセットに含めることを推奨します。
        - ただし、単純なコマンドをただタスク化する場合は、プロンプト等でそのコマンドを直接言及するだけで済む場合も多いです。

## たまに使うかもしれないツール一覧
以下のツールは特定の状況下でのみ使用されるため、必要に応じてツールセットに含めることを推奨します。

- get_changed_files
    - Gitリポジトリの変更点を把握するために有用ですが、大抵の場合Gitコマンドを直接使用して済むことが多いです。
- test_failure
    - テストを実行するプロジェクトで有用ですが、全てのプロジェクトで必要なわけではありません。また、コマンドラインでテスト結果を取得して済む場合も多いです。
- open_simple_browser
    - エディタ内でWebページをプレビューするためのツールで、ローカルのWEBサイト開発時に有用ですが、通常の開発ではあまり必要とされません。本格的にWEBサイトの情報をAIに取得させてテスト・確認等を行わせたい場合は、Playwright等の専用ツールを使用することを検討してください。
- タスク・ターミナル系(terminal_last_command・terminal_selection)
    - どちらも他の手段で代替可能な場合が多いため、特別な理由がない限りツールセットに含める必要はありません。
